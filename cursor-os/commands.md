# Cursor Repo OS Commands

Use these as standard operator commands in Cursor.

## `/start`
Summarize repo purpose, architecture, current state, and top priorities by reading all brain files.

## `/resume`
Continue from `brain/next_steps.md` with focus on `Now`, then `Blocked`.

## `/plan`
Create a concise execution plan for the requested task:
- scope
- assumptions
- ordered steps
- validation strategy

## `/update-brain`
Sync all memory files after work:
- `brain/current_state.md`
- `brain/next_steps.md`
- `brain/decisions.md` (if needed)
- `brain/change_log.md` (append)

## `/analyze`
Re-scan codebase and refresh:
- `brain/project_context.md`
- `brain/architecture.md`
- maturity assessment and unknowns

## `/ship`
Finalize a task with:
- verification summary
- known limitations
- handoff notes
- brain updates complete
